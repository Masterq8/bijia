## 🧑‍💻 开发调试

### 浏览器开发者工具检查

**快捷键：** F12 或 右击 → 检查

**重要标签：**
- **Console**：查看错误日志
- **Network**：查看 API 请求
- **Application** → **Storage** → **Local Storage**：查看 Token

### Console 调试命令

在浏览器 Console 中执行：

```javascript
// 查看已保存的 Token
localStorage.getItem('Authorization')
// 输出: "Bearer eyJhbGc..." 表示已登录

// 清除 Token（模拟登出）
localStorage.removeItem('Authorization')

// 查看所有 localStorage 内容
Object.keys(localStorage).forEach(key => {
  console.log(`${key}: ${localStorage.getItem(key)}`)
})

// 测试后端连接
fetch('http://localhost:8066/captchaImage')
  .then(r => r.json())
  .then(d => console.log('后端可用:', d))
  .catch(e => console.log('后端不可用:', e))
```

### 网络请求监控

在 Network 标签中，可以看到所有 API 请求：

```
请求 1: GET /captchaImage
  状态：200
  响应体：{ code: 200, img: "...", uuid: "..." }

请求 2: POST /login
  请求体：{ username: "admin", password: "123456", code: "1234", uuid: "..." }
  状态：200
  响应体：{ code: 200, token: "eyJhbGc..." }

请求 3: 任何其他请求
  Header: Authorization: Bearer eyJhbGc...
```

## 🔍 常见问题排查

### 问题 1：验证码无法加载（显示空白）

**可能原因：**
- 后端服务未启动
- VITE_APP_BASE_API 配置错误
- 浏览器跨域问题

**排查步骤：**
```
1. 在 Console 中执行：
   fetch('http://localhost:8066/captchaImage')
     .then(r => r.json())
     .then(d => console.log(d))
     
2. 查看是否有错误提示

3. 确认后端 CORS 已配置
```

### 问题 2：输入框无法输入

**可能原因：**
- 浏览器插件干扰
- 页面加载未完成

**解决方案：**
```
1. 刷新页面（Ctrl+R）
2. 清空浏览器缓存（Ctrl+Shift+Delete）
3. 尝试不同浏览器
4. 禁用浏览器插件后重试
```

### 问题 3：登录/注册按钮无反应

**可能原因：**
- 表单校验未通过
- 后端服务未启动
- 请求被阻止

**排查步骤：**
```
1. 检查所有输入框是否填写
2. 打开 Console 看是否有错误
3. 打开 Network 看请求是否发送
4. 检查后端服务状态
```

### 问题 4：页面样式异常

**可能原因：**
- CSS 文件加载失败
- 浏览器缓存问题

**解决方案：**
```
1. Ctrl+Shift+R 硬刷新（清缓存）
2. 关闭浏览器完全重新打开
3. 使用隐私模式打开
```

## 🎓 知识点讲解

如果要给技术新手讲解：

### 讲解点 1：前后端分离
```
演示：打开 Network
说明：前端向后端发送请求，后端返回JSON数据
要点：完全解耦，前端只负责UI和交互
```

### 讲解点 2：Token 认证
```
演示：登录前后对比 Authorization header
说明：登录成功后，每个请求都自动带上 Token
要点：有了 Token，后端就知道你是谁
```

### 讲解点 3：表单校验
```
演示：输入不合法数据看错误提示
说明：前端校验快速反馈，后端校验确保安全
要点：双重校验，提升用户体验和安全性
```
